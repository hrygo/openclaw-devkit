# OPENCLAW ORCHESTRATION: INSTALLATION LOGIC FLOW

本图展示了执行 `make install` 时，系统如何处理环境变量、命令行参数以及最终选择镜像与 Dockerfile 的全过程。

```text
┌────────────────────────────────────────────────────────────────────────┐
│                        USER COMMAND: make install                      │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │
                                    ▼
┌────────────────────────────────────────────────────────────────────────┐
│  PHASE 1: 环境注入 (ENVIRONMENT INJECTION)                              │
│ ---------------------------------------------------------------------- │
│ 1. Makefile 加载 .env 文件                                              │
│ 2. 获取 $OPENCLAW_IMAGE (例如: ghcr.io/hrygo/openclaw-devkit:dev)      │
│ 3. 变量初始化: IMAGE_NAME := $(OPENCLAW_IMAGE) || "openclaw-devkit:dev"│
└───────────────────────────────────┬────────────────────────────────────┘
                                    │
                                    ▼
┌────────────────────────────────────────────────────────────────────────┐
│ PHASE 2: 参数解析与动态派生 (DYNAMIC DERIVATION)                         │
│ ---------------------------------------------------------------------- │
│ 调用 select_image(目标参数) 函数:                                        │
│                                                                        │
│ IF [无参数]:                                                           │
│    IMAGE_NAME := INITIAL_IMAGE_NAME (维持 .env 原样)                   │
│                                                                        │
│ IF [参数=java / office / dev]:                                         │
│    IMAGE_NAME := INITIAL_IMAGE_NAME + "-参数" (例如: ...:latest-office) │
│    [核心特性]: 自动对齐 GHCR 预构建镜像命名规范                          │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │ (确定最终 IMAGE_NAME)
                                    ▼ 
┌────────────────────────────────────────────────────────────────────────┐
│ PHASE 3: 脚本传递 (SCRIPT HANDOVER)                                     │
│ ---------------------------------------------------------------------- │
│ 执行: ./docker-setup.sh --image ${IMAGE_NAME}                          │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │
                                    ▼
┌────────────────────────────────────────────────────────────────────────┐
│ PHASE 4: DOCKERFILE 智能检测 (DOCKERFILE DETECTION)                     │
│ ---------------------------------------------------------------------- │
│ docker-setup.sh 分析 ${IMAGE_NAME} 字符串内容:                           │
│                                                                        │
│ - 包含 "java":   使用 Dockerfile.java                                   │
│ - 包含 "office": 使用 Dockerfile.office                                 │
│ - 默认:          使用 Dockerfile                                        │
└───────────────────────────────────┬────────────────────────────────────┘
                                    │
                                    ▼
┌────────────────────────────────────────────────────────────────────────┐
│ PHASE 5: 执行引擎 (EXECUTION ENGINE)                                    │
│ ---------------------------------------------------------------------- │
│ IF [极速模式/PULL MODE]:                                                │
│    docker pull ${IMAGE_NAME}                                           │
│                                                                        │
│ IF [开发模式/BUILD MODE]:                                               │
│    docker build -f [选定的Dockerfile] -t ${IMAGE_NAME}                  │
└────────────────────────────────────────────────────────────────────────┘
```
